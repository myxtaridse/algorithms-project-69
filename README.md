# Search engine
Поисковый движок - это проект, в котором применяются основные алгоритмы и структуры данных в поисковых системах.

## Requirements
- Unix system (Macos/Linux)
- Nodejs 22.x+
- Make

## Description
В проекте применяются:
- **Инвертированный индекс:** структурированный словарь формата `{ word: [<id_doc>] }`. Повторяющиеся идентификаторы документов внутри массивов используются в качестве счетчика частоты слов в самом тексте документа.
- **Нечеткий поиск:** механизм обработки составных поисковых запросов. Поиск базуируется на определении пересечений множеств: система изолированно извлекает списки документов для каждой подстроки из запроса, а затем агрегирует их, вычисляя релевантность документов, содержащих как полную исходную фразу, так и лишь подмножество ее слов.
- **Ранжирование**
Ранжирование результатов поисковой выдачи основано на вычислении весов соответствия документов по двум независимым критериям:
    - Координационное соответствие (Уникальность): Первичный критерий ранжирования. Измеряет количество уникальных подстрок из поискового запроса, пересекающихся с текстом конкретного документа. Документ, содержащий большее разнообразие искомых слов, получает наивысший приоритет.
    - Суммарная частота терминов (Плоность): Вторичный критерий ранжирования (используется при равенстве координационного соответствия). Оценивает абсолютную частоту вхождения всех совпавших подстрок в документе, отдавая приоритет текстам с наибольшей плотностью упоминания элементов запроса.

## Installation
```bash
git clone https://github.com/myxtaridse/algorithms-project-69.git
cd algorithms-project-69
make install
```

## Usage (Пример использования)
```javascript
import search from '@hexlet/code';

const doc1 = { id: 'doc1', text: "I can't shoot straight unless I've had a pint!" };
const doc2 = { id: 'doc2', text: "Don't shoot shoot shoot that thing at me." };
const doc3 = { id: 'doc3', text: "I'm your shooter." };
const docs = [doc1, doc2, doc3];

search(docs, 'shoot'); // ['doc2', 'doc1']
```
